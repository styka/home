1. Opis projektu — funkcjonalny i techniczny

Wizja & Nazwa

WorldOfMag (wewnętrzna nazwa: Omnia) to osobisty system zarządzania życiem i pracą dla Szymon Tyki. Nazwa oznacza "Świat Maga" — jego cyfrowy wszechświat integrujący wszystkie aspekty życia: zakupy, zadania, notatki, kuchnię, zwierzęta, zdrowie, nawyki, pojazdy, finanse, języki, wiadomości, pogodę, magazynowanie, warsztaty, a także usługi (marketplace klient ↔ wykonawca).

Aplikacja wyrosła z prostego narzędzia na skalę trzech modułów do zaawansowanego systemu operacyjnego dla życia — wszechstronnego, opartego na AI, wspieranego sztuczną inteligencją dla analizy i rekomendacji.

Strategia biznesowa: Droga do 100M użytkowników

Faza 1 (obecnie): 1–50 użytkowników — validation, MVP modules, free tier, focus na core UX.

Faza 2 (po 3–6 miesiącach): 50–10k użytkowników — monetyzacja (ads + premium), marketplace integrations (Gmail, Google Calendar, Slack), mobile app (PWA lub React Native).

Faza 3 (1–2 lata): 10k–1M użytkowników — team collaboration, enterprise features, compliance (SOC 2, GDPR), SaaS white-label dla branż.

Faza 4 (2–5 lat): 1M–100M użytkowników — platforma dla małych biznesów (modułowe aplikacje na życzenie, tanio dzięki AI), ecosystem integracji (API, webhooks), global payments.

Model monetyzacji

  • Free tier (Dom): Wszystkie moduły podstawowe — shopping, tasks, notes, kitchen, pets, health, habits, languages, news, weather, storage (mode Dom), workshops (mode Dom). Reklamach (Google AdSense lub właściwe).
  • Premium (Pro): Advanced features — team collaboration, advanced storage (Pro mode: invoicing, suppliers), workshop management (Pro mode: projects), advanced analytics, no ads. ~$9–15/miesiąc.
  • Marketplace: Wykonawcy usług (Pro subscription required) — float na prowizjach (5–10% per transakcję).

Architektura wysokiego poziomu

Warstwy aplikacji

  1. Warstwy prezentacji (React): Komponenty per moduł (shopping/, tasks/, notes/, ...). ModuleSidebar (desktop) + top bar (mobile) + bottom tab bar. Server Components na granicach. Client Components dla interaktywności (useState, useTransition).
  2. Warstwa biznesowa (Server Actions): ~48 plików, każdy = operacja (createItem, updateTask, deleteNote). Auth-first (requireAuth), access checks, Prisma mutations, revalidatePath.
  3. Warstwa LLM (API Routes): 15+ endpointów pod /api/llm/** — routing providera (Groq/Anthropic/OpenAI), parsing, normalizacja, Q&A, agent conversacyjny.
  4. Warstwa danych (Prisma ORM): 56+ modeli, relationships (1:N, N:N), constraints, indexes. Migracje = source of truth dla schematu.
  5. Warstwa auth (NextAuth): JWT cookie-based, Google OAuth, PrismaAdapter. Session w request context.

UX Philosophy

  • Keyboard-first: vim-style shortcuts (j/k, x, e, d, a/n, /f, 1–5, Ctrl+K, Esc). Zero unnecessary clicks.
  • Dark theme: CSS variables (--bg-base, --text-primary, --accent-blue) — skalowalne skin system (user/team/system skins).
  • Minimalist design: Linear, GitHub, VS Code aesthetic. Spaced out, breathing, readable.
  • Mobile-first responsive: Sidebar hidden (md:hidden), top bar + bottom tab bar on mobile. Safe area insets, touch targets ≥44px.

AI & LLM Integration

Aplikacja używa Groq API (llama3, mixtral) do:

  • Parsing tekstu wolnego: "2 opakowania mleka" → qty=2, unit="opakowań", name="mleko" (Shopping, Tasks).
  • Sugestie i rewrite: Sugestie tagów notatek, przeformułowanie, korekta grammatyczna (Notes).
  • Semantyczne wyszukiwanie: "Gdzie przechowuję śrubki?" → Storage (semantic search).
  • OCR & Vision: Skan faktur, zdjęcia spisu magazynu (Storage Pro, Kitchen OCR).
  • Agent conversacyjny: FAB "Sparkles" asystenta AI — robi plany, wyszukuje, wydaje komendy (Home module).
  • Web-search: Brave API → DDG fallback (News module).

Model routing: Per operation type (reasoning, dispatch, thinking, images, generation) — konfiguracja w /admin/llm.
